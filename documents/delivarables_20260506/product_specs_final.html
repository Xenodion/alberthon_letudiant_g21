Product 01
High-Intent Lead Feed
From a static contact list to a dynamic, ranked signal of purchase intent
Client
Grandes écoles, universities, bootcamps — institutions with a defined cost-per-acquisition budget and a recruitment team that buys contact lists today
Problem
L'Étudiant currently sells undifferentiated contact lists: a student who attended a salon last week and a student who registered two years ago cost the same per lead. The client pays for cold leads without knowing it.
Description
Weekly export of the top 200 prospects, re-ranked in real time from recent behavioural signals. The product is not the profile — it is the rank. A school receives a list sorted by who is most likely to respond to a contact attempt right now.
Inputs
(BigQuery)
  • Salons_Inscrits_et_venus — salon registration dates, attendance, study level
  • Site_Inscrits — profile completeness, opt-ins, geography, age
  • CRM_Communication — email opens and clicks (season 25/26)
  • Agent_Conversationnel_ORI_Conversation — chatbot sessions, engagement depth
Transformations
  • Freshness score (35%): exponential decay λ=0.015 from last cross-table interaction — half-life ~46 days
  • Intent score (45%): weighted signals (salon registration ×20, attendance ×10, chatbot ×15, email click ×12, email open ×5) — tanh-normalised
  • Completeness score (20%): email, phone, region, age, study level, field of study — each field weighted by commercial usability
  • GDPR compliance filter: optin_commercial_actuel = True · age ≥ 18 · ACTIF = Toujours inscrit
  • Percentile classification: top 15% → Class A, next 30% → Class B, bottom 55% → Class C
Output
Weekly CSV or API response: pseudonymised ID, composite score, class (A/B/C), freshness score, intent score, completeness score, life-moment segment, recency bucket, study level, region. No PII in output.
Pricing
€0.80 – €2.00 / scored lead
vs. €0.30–0.60 for a raw undifferentiated list → ×3 revenue per lead potential
Alternative: bundled into Premium Access subscription (€2,500/month) for weekly delivery
Gross Margin
85%
Marginal cost ~€0.08/lead · infrastructure shared across all products · no variable data acquisition cost
Product 02
No-Show Intelligence Report
Transforming a sunk cost into a re-engagement opportunity
Client
Salon exhibitors — schools and institutions that pay for a booth at L'Étudiant's fairs and want to identify registered students who did not show up and remain contactable
Problem
43% of registered students do not attend the salon they signed up for — observed directly on real BigQuery data across multiple seasons. This is not an estimate. Each no-show expressed purchase intent (they registered) and then disappeared. Exhibitors currently have no visibility into who did not come, nor any mechanism to re-engage them. The data exists; it has never been turned into a product.
Description
A post-salon report delivered within 48 hours of each fair's closing. It identifies which registered students did not attend, segments them by profile and study level, flags those with prior CRM engagement (indicating higher re-engagement potential), and ranks them by composite score for immediate outreach.
Inputs
(BigQuery)
  • Salons_Inscrits_et_venus — Showed_up = False, salon date, registration date, study level
  • Site_Inscrits — optin_commercial_actuel, region, acquisition source, age
  • CRM_Communication — prior email engagement before the salon date (signal of pre-salon intent)
Transformations
  • No-show filter: Showed_up = False AND salon registration date IS NOT NULL AND id_Inscrit_site IS NOT NULL
  • GDPR compliance filter: same three mandatory checks as Product 1
  • Segmentation: by study level (Terminale, Bac+1…), by acquisition source, by region
  • Re-engagement score: email opened or clicked in the 30 days before the salon + chatbot session = high re-engagement priority
  • Ranking: composite score applied to no-show subset — highest-scored no-shows surfaced first
Output
PDF report (narrative + tables) + CSV export: no-show count by segment, re-engagement score distribution, top 50 re-engageable profiles ranked by composite score. Delivered within 48h of salon closing.
Pricing
€3,000 – €8,000 / salon edition
Price scales with salon size (registrant count). 10 salons/year × 25% exhibitor adoption in Y1 → €12,500 ARR. At 55% adoption (Y3) → €27,500 ARR.
Gross Margin
93%
Fully automated pipeline — cost is compute + QA review only · no manual data preparation
Product 03
Life-moment Audience API
Self-service segmentation for financial, housing and mobility partners
Client
Financial partners (Boursorama, Hello Bank), student housing platforms (Studapart, NEXITY), insurers, international mobility operators — any B2B partner that needs to reach students at a specific life transition moment
Problem
Today, a bank wanting to reach students who have opted in to financial products must call L'Étudiant's commercial team, negotiate a list, wait for manual extraction, and receive a CSV with no consent documentation. The process takes days and does not scale. There is no self-service, no audit trail, and no way to target by life-moment rather than demographic criteria alone.
Description
A REST API endpoint that lets partner companies query L'Étudiant's database by life-moment segment in real time. A query for financial_entry returns profiles of students who have explicitly opted in to financial communication and are entering higher education. Consent audit trail is built into every response.
Life-moment
Segments
  • financial_entry — optin_FINANCE = True or study domain contains finance/banking. Target: banks, insurers.
  • housing_transition — optin_HOUSING = True or study domain contains housing/real estate. Target: student housing platforms.
  • coaching — optin_COACHING = True. Target: orientation advisors, bootcamps.
  • orientation_decision — secondary school level or active chatbot sessions. Target: grandes écoles, universities.
Inputs
(BigQuery)
  • Site_Inscrits — optin_FINANCE, optin_HOUSING, optin_COACHING, id_domaine_etude, region
  • Site_Inscrits_dimension_domaine_etude — study domain labels for keyword classification
  • All behavioural tables — for composite score applied to segment
Transformations
  • Life-moment classification: opt-in signals evaluated first (explicit consent > inferred intent); school level and chatbot as fallback signals
  • GDPR compliance filter: same three mandatory checks; consent type recorded per profile per export
  • Scoring: composite A/B/C applied within each segment — caller can filter by class
  • Audit trail: each API response logs export timestamp, partner ID, segment queried, optin_type, profile count — immutable record for DPO
Output
JSON response per profile: pseudonymised ID, life-moment segment, lead class (A/B/C), composite score, region, study level. No PII. Consent type and audit timestamp included in response header.
Pricing
€500 – €2,000 / month platform access
+ €0.50 – €1.20 / exported profile
Tier: Standard API at €500/month for up to 500 profiles/month · Premium API at €2,000/month for unlimited with SLA. Usage-based overage above monthly threshold.
Gross Margin
78%
Lower than P1/P2 due to API infrastructure cost + DPO audit overhead + ongoing partner management

GDPR Compliance Note — Pipeline by Design

No profile reaches an activation output without passing all three compliance filters. These checks run automatically at every pipeline execution, before any export or API response is generated.

✓optin_commercial_actuel = True — explicit commercial consent required. Profiles without this field set to True are excluded from all outputs regardless of score.
✓Age verification on Naissance_Date — profiles where date of birth indicates the individual is under 18 are automatically excluded. Benefit of the doubt applies where date of birth is absent.
✓ACTIF = Toujours inscrit — users who have unsubscribed or deactivated are excluded. Deactivation propagates to all pipeline outputs via the id_Inscrit_site key.
✓No PII in any output — email, phone number and name are never included in scored profiles, exports or API responses. Only the pseudonymised id_Inscrit_site is carried through.
✓Right to erasure — the id_Inscrit_site key enables cascading deletion across all pipeline outputs: scored CSVs, activation CSVs, and barometer JSON are all keyed on this single identifier.
✓Consent audit trail (Product 3) — every Life-moment API export logs partner ID, segment queried, export timestamp and optin_type. Immutable record for DPO compliance review.
